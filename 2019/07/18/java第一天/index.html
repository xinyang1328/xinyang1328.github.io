修饰符

访问控制修饰符

默认的，在同一个文件内除了子类都阔以访问嘞

公用的，权利最留批的一个，阔以跨文件访问

受保护的，在同一个文件都阔以滴，子类也可以访问。

私有滴，只能自身访问，其他的包括子类都是访问不到滴

所以，被访问范围为：

public > protected > default > private

其他修饰符：

声明独立于对象的静态变量或静态方法。

定义一个变量的时候，和js里面的const差不多，都是使变量的值不可改变；
定义一个函数的时候，没有类可以继承final类的任何特性

我个人感觉有点类似c++里面的虚函数，在父类里面定义一个函数，然后再子类里面进行定义这个函数的作用，不同的子类的函数互不影响。

被声明的方法只能被一个线程访问。

可以让变量反序列化。(至于序列化是个啥…我也有点懵逼，就算是辣个留批的后台人，也无法解决我的疑惑

修饰的成员变量在每次被线程访问时，都强制从共享内存中重新读取该成员变量的值。而且，当成员变量发生变化时，会强制线程将变化值回写到共享内存。这样在任何时刻，两个不同的线程总是看到某个成员变量的同一个值。
